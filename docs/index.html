Open-source firmware, a phone app, and Home Assistant integration for an ESP-based clock — automatic time, adaptive brightness, and presence sensing, all running locally. Built and proven on the Ajanta OLC-501, and adaptable to similar displays.

⚡ Flash it from your browser

No toolchain, no drivers to wrestle. Plug your board into your computer with a USB cable, pick it below, and click connect. About a minute, start to finish.


Your browser can't flash — please use desktop Chrome or Edge. You'll need to allow access to the serial port to continue.
Desktop Chrome or Edge only (they support Web Serial). After flashing, connect to the Clockwise-Setup WiFi the clock creates to finish setup. Prefer the command line? Build it with PlatformIO.

What it does

🕐

Always-right time

WiFi + NTP with a real-time-clock backup through power cuts. Handles daylight saving itself.

🔆

Adaptive brightness

Manual, a day/night schedule, or a Sun mode that ramps with real sunrise & sunset for your location.

👋

Presence sensing

An optional 24 GHz mmWave radar detects presence and distance, and can dim the display when the room's empty.

📱

Phone app

An Android app auto-discovers the clock on your network for full control and sensor tuning.

🏠

Home Assistant

MQTT auto-discovery builds the whole device automatically, plus a custom clock-face card.

🔒

Local & secure

Runs entirely on the clock — no cloud. PIN-protected API and authenticated over-the-air updates.

After flashing

  1. Connect to the Clockwise-Setup WiFi the clock broadcasts.
    A setup page opens automatically.
  2. Pick your home WiFi, enter its password, set an optional PIN.
    Your timezone is detected in the browser.
  3. Install the app, or add it to Home Assistant — and you're done.

Using a different clock or board?

Clockwise is built for the Ajanta OLC-501, but the firmware, app, and Home Assistant pieces are a clean reference for any ESP 7-segment clock. Support for more boards is planned — the flasher above is built to add them.
